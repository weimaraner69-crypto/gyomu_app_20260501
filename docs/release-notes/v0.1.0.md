# v0.1.0 リリースノート

リリース日: 2026-05-01

## 概要

社内向け出退勤管理アプリの初回リリースです。

- メールログイン
- 出勤・休憩入り・休憩戻り・退勤打刻
- 日報提出
- 管理者向け月間サマリー/打刻履歴

## 主な追加機能

### 1. 認証と画面導線

- ログイン画面を追加
- 未ログイン時の保護ページアクセス制御
- ログアウト API を追加

### 2. スタッフ向け機能

- 出勤/退勤打刻
- 休憩入り/休憩戻り打刻
- 当日日報の作成・提出

### 3. 管理者向け機能

- 月間サマリー表示
- 打刻履歴テーブル表示

### 4. DB と権限設定

- `profiles` / `attendance` / `daily_reports` テーブル
- RLS ポリシー
- 新規ユーザー登録時の `profiles` 自動作成トリガー

## 重要な修正

### RLS 再帰エラー修正

`profiles` の管理者ポリシーで発生する再帰エラーを修正しました。

- 管理者判定関数 `public.is_admin()` を導入
- 管理者向け SELECT ポリシーを関数経由へ変更

### 管理画面の取得不具合修正

`attendance` と `profiles` の取得方法を見直し、管理画面の一覧表示が空になる問題を修正しました。

## 仕様変更（休憩時間）

休憩時間を「60分固定」から「実績記録」に変更しました。

- `休憩入り` で `break_started_at` を記録
- `休憩戻り` で実績分を `break_minutes` に加算
- 退勤時の勤務時間は次で算出

`work_minutes = max(0, (clock_out - clock_in) - break_minutes)`

## 既存環境のアップデート手順

既に運用中の DB には、以下の順で SQL を適用してください。

1. `supabase-hotfix-rls-recursion.sql`
2. `supabase-hotfix-break-tracking.sql`

## 既知の注意点

- 休憩時間は分単位で計算されます（秒は切り捨て）。
- 初期導入時は管理者ユーザーの `profiles.role='admin'` 設定が必要です。
